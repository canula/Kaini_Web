Getting Started

TC-01 Tone Calculator is an algorithmic melody generator available as a free web app, Android app, and desktop application. It generates melodies by combining eight independent musical parameters — key, scale, intervallic motion, contour, rhythm, ornamentation, instrument, and phrase structure — each controlled through a dedicated transform (TR1–TR8).

Core Capabilities

  • - Generate melodies across any key, scale, or mode with configurable interval and contour behaviour
  • - Build 8-bar phrases automatically using compositional form templates (ABA, AABA, Rondo, etc.)
  • - Import MIDI files or record vocals for pitch-detected fragment extraction
  • - Edit individual notes in a tracker-style editor Pro
  • - Export to MIDI, WAV, or the native .kai preset format Pro
  • - Organize compositions into named sections with independent bar configurations

Two Composition Modes

Phrase Mode uses Bar 1 and Bar 2 as seed material, then applies a form template and five development parameters to generate Bars 3–8 automatically. Free Mode provides direct control over every transform in every bar, allowing fully manual composition. Both modes are documented in detail below.

Phrase Mode

Phrase Mode generates complete 8-bar compositions from two bars of seed material. It takes Bar 1 and Bar 2 as input, applies a compositional form template (ABA, AABA, Rondo, etc.), and algorithmically generates Bars 3–8 based on five development parameters. The result is a structurally coherent phrase that follows established compositional conventions.

How It Works

  1. 1.
    Enable Phrase Mode — Toggle in the transport bar (dual-purpose button: shows "Phrase" when Select Mode is off). This activates TR8 and locks Bars 3–8 to algorithmic generation.
  2. 2.
    Open TR8 (Phrase Transform) — Opens the Phrase Mode modal containing 100+ presets and five development sliders.
  3. 3.
    Select a Preset — Each preset defines a form template, slider defaults, and recommended scale/motion settings. Selecting a preset populates Bar 1, Bar 2, and all five sliders. You can then override Bar 2 or adjust any slider before generating.
  4. 4.
    Press Play — The algorithm generates Bars 3–8 from the seed material and plays the full 8-bar phrase.

Development Parameters

These five sliders in the TR8 modal control how the algorithm develops seed material across Bars 3–8:

Motivic Return

Controls how frequently the Bar 1 motif recurs in later bars. Low values produce more new material in each bar. High values create frequent callbacks to the opening idea, reinforcing thematic unity.

Development Intensity

Degree of variation applied to middle bars (typically Bars 3–6). Low values make subtle changes — minor rhythmic displacement, small intervallic shifts. High values apply significant rhythmic and intervallic transformation to the source material.

Energy Direction

Shapes the dynamic arc of the phrase. "Build" creates progressively more active material (increasing note density, wider intervals, higher register). "Wind Down" does the inverse — progressively less active material toward Bar 8.

Resolution Strength

Determines how definitively the phrase concludes at Bar 8. Low values leave the ending open — unresolved intervals, sustained motion. High values produce strong cadential resolution — stepwise motion to the tonic, rhythmic deceleration.

Harmonic Motion

Degree of key and scale modulation across the 8 bars. Low values keep the entire phrase in the original key. High values introduce modulation to related keys in the middle bars before returning for resolution.

Editing Seed Material

Bar 1 and Bar 2 remain fully editable in the main UI while Phrase Mode is active. Any changes to their transform settings (key, scale, motion, contour, rhythm, instrument) are reflected in the generated output. Presets populate both bars on selection, so select a preset first, then customise Bar 2 to introduce variation in the seed material.

Free Mode

Free Mode provides direct control over all transform parameters in every bar. Each bar is configured independently, and TR8 (Phrase) is disabled. This is the default composition mode.

Workflow

  1. 1.
    Select Bar 1 — Bar 1 is active and selected by default.
  2. 2.
    Configure transforms:
    • - TR1 (Key): Set the tonal centre
    • - TR2 (Scale): Define the available pitch set
    • - TR3 (Motion): Set the interval distribution (Stepwise, Skip, Leap, Mixed)
    • - TR4 (Contour): Set the melodic direction (Rising, Falling, Arch, Valley, Plateau)
    • - TR5 (Rhythm): Select or build a duration pattern
    • - TR7 (Instrument): Choose sound source and audio parameters
  3. 3.
    Duplicate to Bar 2 — Drag Bar 1 onto Bar 2 to copy all transform settings. This creates an identical bar that can be modified selectively.
  4. 4.
    Modify Bar 2 — Change one or two parameters (e.g., switch Motion from Stepwise to Skip, or Contour from Arch to Valley) to create variation while retaining shared characteristics.
  5. 5.
    Play and extend — Listen to the result, then continue adding bars. Each bar can share settings with previous bars or diverge entirely.
Composition Patterns

Call-and-Response (2 Bars)

  • - Bar 1: Rising contour, Stepwise motion — establishes a melodic idea with ascending tension
  • - Bar 2: Falling contour, Skip motion — responds with descending resolution and wider intervals

Motivic Development (4 Bars)

  • - Bar 1: Original motif (e.g., C Major, Arch contour, Stepwise)
  • - Bar 2: Same key/scale, different rhythm — rhythmic variation of the motif
  • - Bar 3: Same rhythm as Bar 2, different contour — contour variation
  • - Bar 4: Return to Bar 1's rhythm in a new key (e.g., G Major) — transposed recapitulation

Contour Sequencing (4 Bars)

  • - Bar 1: Rising — ascending pitch trajectory
  • - Bar 2: Falling — descending pitch trajectory
  • - Bar 3: Arch — ascent followed by descent within a single bar
  • - Bar 4: Plateau — minimal pitch variation, rhythmic interest

The Bar System

Each section contains 8 bars. Every bar holds an independent set of transform settings (TR1–TR7) and can be in one of four states:

Active

Included in playback

Inactive

Settings retained, excluded from playback

Selected

Currently targeted for editing

Playing

Currently sounding during playback

Bar Operations

  • Tap/Click a bar — Selects it. Opening any TR modal now edits that bar's settings.
  • Drag a bar onto another bar — Copies all transform settings (TR1–TR7) from the source bar to the target. The target bar's previous settings are overwritten. The source bar is unchanged.
  • Drag a bar outside the grid — A bin icon appears with a red flash. Releasing the drag deletes the bar's settings and sets it to inactive.

Playback Rules

  • - Default: Plays sequentially from Bar 1 through the last active bar in the section
  • - With marked bars (Select Mode): Only bars with an orange border participate in playback
  • - Each bar generates its melody independently from its own transform settings

The Section Builder

Sections are 8-bar containers that play sequentially in the order they appear in the carousel. Each section holds its own set of bar configurations and can be named for organisational clarity (e.g., "Verse", "Chorus", "Bridge").

Each section contains:

  • - 8 bars, each with independent transform settings
  • - A user-assignable name
  • - Independent active/inactive state per bar
Creating, Renaming & Deleting Sections

Creating:

  1. 1. Click/Tap the "+" button in the section carousel
  2. 2. A new section appears with a default name ("Section [number]") and all 8 bars inactive
  3. 3. Drag one section onto another to copy all bar configurations (all TR1–TR7 settings for all 8 bars)
  4. 4. Activate bars individually by clicking/tapping them

Renaming:

  1. 1. Select the section in the carousel
  2. 2. Open Hamburger Menu > Rename Current Section
  3. 3. Enter the new name

Deleting:

Drag the section chip outside the carousel area — a bin icon appears with a red pulse — release to delete. The last remaining section cannot be deleted. Deletion is permanent.

Playback Across Sections
  • - Sections play in sequential order (left to right in the carousel)
  • - Playback advances automatically from the last active bar of one section to Bar 1 of the next
  • - The carousel auto-scrolls to the currently playing section (green glow)
  • - The currently selected section is highlighted with an orange border
  • - Navigate between sections using the chevron buttons (< >) or by swiping/scrolling the carousel

Select Mode

Select Mode lets you mark specific bars or sections for focused playback. Enable it via the "Select" button in the transport controls.

When Select Mode is ON:

  • - Tap/Click any active bar or section to toggle its "marked" state (orange border)
  • - The dual-purpose button switches to Loop mode (for continuous repeat playback)
  • - Only marked bars/sections will play

When Select Mode is OFF:

  • - Normal tap behavior (select to view and edit transforms)
  • - The dual-purpose button shows Phrase Mode toggle

Important: Select Mode is disabled (grayed out) when Phrase Mode is active. Enabling Phrase Mode automatically clears all marked bars and disables Select Mode.

Transform Reference: TR1-TR2 (Key & Scale)

TR1: Key Transform

Sets the tonal centre for the bar. All pitch generation is relative to this root note. Changing the key transposes the entire melodic output while preserving interval relationships.

Options: C, C♯/D♭, D, D♯/E♭, E, F, F♯/G♭, G, G♯/A♭, A, A♯/B♭, B

TR2: Scale Transform

Defines the pitch set available for melody generation within the selected key. The scale determines which intervals are used and constrains all generated notes to its member pitches.

Common Scales

Major (Ionian): 7-note diatonic scale with major 3rd and major 7th. W-W-H-W-W-W-H interval pattern.

Natural Minor (Aeolian): 7-note diatonic scale with minor 3rd and minor 7th. W-H-W-W-H-W-W interval pattern.

Harmonic Minor: Natural minor with a raised 7th degree, creating an augmented 2nd between degrees 6 and 7.

Melodic Minor: Raised 6th and 7th degrees ascending; reverts to natural minor descending.

Modes

Dorian: Minor mode with a raised 6th degree. Common in jazz and folk idioms.

Phrygian: Minor mode with a lowered 2nd degree. Characteristic half-step from root.

Lydian: Major mode with a raised 4th degree. The augmented 4th creates a tritone from root.

Mixolydian: Major mode with a lowered 7th degree. Dominant-quality scale.

Aeolian: Identical to natural minor. Minor 3rd, minor 6th, minor 7th.

Locrian: Diminished mode with a lowered 2nd and 5th. Contains a diminished 5th from root.

Transform Reference: TR3-TR4 (Motion & Contour)

TR3: Motion Transform

Controls the interval distribution between consecutive notes. Each option biases the generator toward a specific interval range while still allowing variation.

Stepwise (50%+)

Biases toward 2nds and 3rds. Over half the intervals will be conjunct (adjacent scale degrees).

Skip (50%+)

Biases toward 4ths, 5ths, and 6ths. Produces wider intervallic movement than Stepwise.

Leap (50%+)

Biases toward 7ths, octaves, and wider. Produces angular, disjunct melodic lines.

Mixed

No bias — intervals are distributed across the full range. Produces varied, less predictable lines.

TR4: Contour Transform

Defines the overall pitch trajectory within a bar. The contour constrains the direction of motion while TR3 determines the interval sizes.

Rising

Net ascending pitch trajectory across the bar. Intervals are biased upward.

Falling

Net descending pitch trajectory across the bar. Intervals are biased downward.

Arch

Ascends in the first half, descends in the second half. Creates a self-contained arc within the bar.

Valley

Descends in the first half, ascends in the second half. Inverse of Arch.

Plateau

Minimal pitch variation. Notes remain within a narrow register, with rhythm providing the primary interest.

Advanced Contour Features

Gravity Points: Define specific pitch targets within the bar. The generator biases note selection toward these pitches at their specified positions, creating intentional focal points in the melodic line.

Multi-Bar Contours: Extend a single contour shape across up to 4 consecutive bars. The contour arc spans the full multi-bar range rather than resetting at each bar boundary.

Transform Reference: TR5-TR6 (Rhythm & Ornaments)

TR5: Rhythm System

Defines the duration pattern applied to generated notes. Rhythm determines note count, timing, and beat placement independently of pitch.

  • - Preset Rhythms: Pre-defined patterns including straight subdivisions (quarter, eighth, sixteenth notes), syncopated patterns (off-beat emphasis), and compound patterns (dotted rhythms, triplets).
  • - Custom Pattern Builder: Construct patterns by toggling individual beat subdivisions on a grid. Each beat can be divided into up to 4 subdivisions. Active subdivisions receive a note; inactive subdivisions are rests or sustains. The resulting pattern is applied cyclically to fill the bar.
  • - Bar Length (Beats Per Bar): Sets the time signature denominator. Options:
    • 2 beats per bar (2/4 time)
    • 3 beats per bar (3/4 time)
    • 4 beats per bar (4/4 time)
    Each bar can use a different beat count, allowing polymetre across bars.
  • - Anacrusis (Pickup Notes): Places one or more notes before the first downbeat of the bar. The pickup notes occupy time borrowed from the preceding bar's final beat. This shifts the melodic phrase to begin ahead of the barline, creating forward momentum into the downbeat.
  • - Preview: Auditions the rhythm pattern with metronome click before applying it to the bar.

TR6: Ornamentation

Applies melodic embellishments to individual notes. Ornaments are inserted at note positions and subdivide the original note's duration to accommodate the additional pitches. Each ornament type has specific rules for when it is applied:

  • - Trill: Rapid alternation between the written note and its upper neighbour (next scale degree). Applied to notes with sufficient duration to accommodate multiple alternations. Subdivides the note into equal-length oscillations.
  • - Mordent: A three-note figure: main note → lower neighbour → main note. Occupies the first portion of the original note's duration, with the remainder sustained on the main pitch.
  • - Turn: A four-note figure that circles the main note: upper neighbour → main note → lower neighbour → main note. Applied to notes long enough for four subdivisions.
  • - Appoggiatura: An accented non-chord tone placed on the beat, resolving stepwise down to the main note. Takes approximately half the original note's duration.
  • - Grace Note: A short, unaccented note preceding the main note. Borrows minimal time from the main note and is played as quickly as possible before the beat.

Transform Reference: TR7 (Instrument)

TR7: Instrument Transform

Selects the synthesis voice and configures audio parameters for the selected bar. Each bar can use a different instrument, allowing timbral variation across the composition.

  • - Instrument: Built-in voices include Piano, Bass, and others. Custom sample-based instruments are available with a Pro license.
  • - Octave Shift: Transposes all notes in the bar up or down by whole octaves without changing the key or scale.
  • - Volume: Per-bar amplitude level (0–100%).
  • - Reverb: Convolution reverb amount (0–100%). Controls the wet/dry mix of the spatial effect.
  • - Stereo Pan: Stereo field position. L = hard left, C = centre, R = hard right.
Custom Instruments Pro

Load any WAV or MP3 file as a custom instrument. TC-01 maps the sample across the pitch range using pitch-shifting, making it available as a playable voice in any bar.

Transform Reference: TR8 (Phrase Mode)

TR8: Phrase Transform

Active only when Phrase Mode is enabled. TR8 controls the algorithmic generation of Bars 3–8 from Bar 1 and Bar 2 seed material. The generation process works as follows: a form template assigns each of the 8 bar positions a structural role (statement, development, recapitulation, etc.), and the five development sliders control how the seed material is varied to fill those roles.

Form Templates

Each preset includes a form template that determines the structural layout of the 8 bars. Common templates:

  • - ABA: Statement (Bars 1–2), Contrast (Bars 3–4), Return (Bars 5–6), Coda (Bars 7–8)
  • - AABA: Statement (1–2), Repeat (3–4), Bridge/Contrast (5–6), Return (7–8)
  • - Rondo: Theme (1–2), Episode 1 (3–4), Theme Return (5–6), Episode 2 / Coda (7–8)
  • - Through-composed: Continuous development with no literal repeats. Each bar pair introduces new material derived from the seed.

Development Parameters

See the Phrase Mode section for detailed documentation of all five sliders (Motivic Return, Development Intensity, Energy Direction, Resolution Strength, Harmonic Motion).

Preset / DNA System

Each of the 100+ presets encodes a complete generation configuration: a form template, default values for all five sliders, and recommended TR1–TR6 settings for Bar 1 and Bar 2. Selecting a preset populates all of these values. You can then override any setting — adjust a slider, change Bar 2's scale, switch the rhythm pattern — and the algorithm regenerates accordingly.

Preset Categories

Classical: Presets based on Western art music conventions. Baroque presets use binary/ternary forms with ornamental variation; Classical-era presets emphasise balanced phrase structure (antecedent-consequent); Romantic presets allow wider harmonic motion and higher development intensity; Minimalist presets use high motivic return with subtle variation.

Jazz: Presets reflecting jazz harmonic and melodic conventions. Bebop presets use chromatic passing tones and high development intensity; Cool Jazz presets favour stepwise motion and moderate energy; Modal presets emphasise scale exploration with low harmonic motion (staying within a single mode).

World: Presets derived from non-Western melodic systems. These use specific scale types (pentatonic, raga-based, gamelan pelog/slendro) and contour patterns characteristic of their source traditions.

Electronic: Presets oriented toward electronic music production. Ambient presets use low development intensity and plateau contours; Sequence presets use repetitive patterns with gradual parameter shifts; Glitch presets apply high development intensity with irregular rhythms.

Experimental: Presets exploring non-traditional pitch organisation. Atonal presets avoid tonal centres; Aleatoric presets maximise randomness across parameters; Micropolyphony presets layer dense stepwise motion.

MIDI Import

Import standard MIDI files (.mid) to extract melodic fragments and their musical properties. Imported material can be analysed, segmented, and applied to bars as transform settings.

Import Process

  1. 1.
    Load a MIDI File — Select a .mid file from your device via "Load from file".
  2. 2.
    Track Selection — Single-track files import automatically. Multi-track files present a track selector to choose which melodic line to analyse.
  3. 3.
    Fragment Analysis — TC-01 segments the MIDI data into bar-length fragments displayed on a pad grid. The analysis detects phrase boundaries based on rests and rhythmic grouping.
  4. 4.
    Property Extraction — For each fragment, TC-01 extracts: the most likely key (TR1) and scale (TR2) from pitch class distribution; the predominant interval motion type (TR3) from consecutive note analysis; the contour shape (TR4) from pitch trajectory; and the rhythm pattern (TR5) from note onset/duration data. These extracted properties can be applied directly to any bar as transform settings.
Managing MIDI Files
  • - Imported files are listed with filename, upload timestamp, and track count badge
  • - Individual files can be deleted with confirmation
  • - Fragment browsing is available after at least one file is imported
  • - All imported files are associated with the current project

Voice Recording

Records monophonic audio (humming, singing, or whistling) and converts it to MIDI note data via pitch detection. Audio files can also be imported instead of recording live.

Process

  1. 1.
    Select Voice Preset — Choose a vocal range (Soprano, Alto, Tenor, Bass, etc.). This constrains the pitch detection algorithm to the expected frequency range, improving accuracy.
  2. 2.
    Record — An optional metronome count-in syncs to the project BPM. A real-time waveform display shows input levels during recording.
  3. 3.
    Pitch Detection — The audio is analysed to extract fundamental frequency per time window. Each detected pitch is mapped to the nearest chromatic note, producing a sequence of MIDI note events with onset times and durations.
  4. 4.
    Quantization — Detected note onsets are snapped to the nearest beat subdivision grid (based on the project BPM and time signature). Note durations are rounded to the nearest grid value. The quantized notes become usable melody fragments that can be applied to bars.
Detection Confidence

Each detected note displays a confidence indicator reflecting pitch detection certainty:

  • - Green (95%+) — High confidence: clear fundamental frequency detected
  • - Amber (90–95%) — Moderate confidence: likely correct but may benefit from manual review
  • - Red (<90%) — Low confidence: ambiguous pitch, manual correction recommended

Audio is processed entirely on-device and is never transmitted externally. Recordings are stored temporarily with a 24-hour expiry.

Note Edit Mode

A tracker-style note editor that lets you modify individual notes within a bar. Open it by double-clicking/double-tapping any active bar.

Features

  • Two Edit Modes:
    • - Chromatic: Edit notes using all 12 semitones
    • - Scale-Relative: Edit notes constrained to the current bar's scale
  • Undo/Redo: Per-bar edit history (up to 16 steps). History resets when switching bars.
  • Preview: Audition your edited notes in real time.
  • Microtonal Support: Available when experimental microtonal mode is enabled.
Pro Feature: Note Edit Mode requires a Pro license. Close the editor with ESC or by clicking outside the modal.

Exporting

TC-01 supports three export formats. Access export options from the Hamburger Menu.

MIDI (.mid)

Standard MIDI file for use in DAWs (Ableton, Logic, FL Studio, etc.). Contains note data, timing, and velocity. Use for further editing, arrangement, and mixing.

Audio (.wav)

High-quality WAV audio file. Direct recording of synth output with all effects applied. Use for sharing, sampling, or archiving.

Preset (.kai)

TC-01 project file (custom format). Contains all transform settings, bar configurations, and sections. Use for saving projects, sharing templates, or building a preset library.

Export Scope Options:

  • - Current Playback: Respects Loop/Phrase Mode/Play Range
  • - All Active Bars: Exports everything marked as active (ignores playback state)

Pro Feature: MIDI and WAV export require a Pro license. Preset (.kai) save/load also requires Pro.

Usage Notes

  • Metronome

    Enable the metronome during playback to verify rhythm alignment, particularly when using syncopated or custom patterns.

  • Rhythm Preview

    TR5 includes a preview function that plays the selected rhythm pattern with a metronome click before applying it to the bar.

  • Polymetre

    Bars can use different beat counts (TR5). Setting Bar 1 to 4/4 and Bar 2 to 3/4 produces polymetric interaction across bars.

  • Ornamentation Placement

    Ornaments (TR6) are most effective on notes with sufficient duration. Applying ornaments to very short notes can produce inaudible subdivisions.

  • Contour + Motion Interaction

    TR3 and TR4 interact: Arch contour with Stepwise motion produces smooth conjunct arcs; Plateau contour with Leap motion produces angular lines within a narrow register.

  • Phrase Mode Development

    To expand a 2-bar idea, configure Bars 1–2 in Free Mode, then enable Phrase Mode and open TR8 to generate Bars 3–8 algorithmically.
