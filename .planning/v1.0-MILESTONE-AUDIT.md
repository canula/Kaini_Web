### GLRY-04: Horizontal scroll/carousel on mobile for phone mockups

**Requirement text:** "Horizontal scroll/carousel on mobile for phone mockups"
**Assigned phase:** Phase 2
**What happened:**
1. Phase 2 Plan 02 built a CSS snap-scroll carousel with 7 phone screenshots — commit `497cb5a`
2. Phase 2 VERIFICATION marked GLRY-04 as SATISFIED
3. Phase 3 Plan 01 replaced the features grid AND gallery carousel with paired alternating feature/image rows — commit `093aa24`
4. The carousel no longer exists in the final HTML. Phone screenshots are now displayed in vertically-stacked feature rows on mobile

**Why this is a gap:** The carousel was correctly built and verified in Phase 2, but Phase 3's restructure removed it. Phase 3's verification scope covered only APP-03 and PRO-05, so GLRY-04 was not re-checked. REQUIREMENTS.md checkbox still shows `[x]` Complete.

**Resolution options:**
1. **Accept as intentional design change** — update REQUIREMENTS.md to reflect the new layout and formally close GLRY-04 as superseded by Phase 3's paired rows
2. **Restore carousel** — add a phone screenshot carousel below the paired feature rows
3. **Add mobile horizontal scroll** — implement snap-scroll behavior within the feature section for mobile viewports
